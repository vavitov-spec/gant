# Настройка вкладки «Аккаунт» — телефон и фото

Во вкладке **Настройки → Аккаунт** есть поля **Телефон** и **Фото**. Они хранятся
в таблице `profiles`, но нужных колонок там ещё нет — их нужно один раз добавить.

Остальное на вкладке (ID участника, ФИО, почта-логин, смена пароля, компании)
работает **без** этого шага. Пока колонок нет — при сохранении телефон и фото
не запишутся, ФИО сохранится, а приложение подскажет про этот файл.

## Что сделать (один раз)

1. Откройте панель Supabase → **SQL Editor**.
2. Вставьте и выполните запрос ниже (он безопасный и аддитивный — существующие
   данные не трогает; повторный запуск ничего не сломает):

```sql
alter table public.profiles add column if not exists phone  text;
alter table public.profiles add column if not exists avatar text;
```

3. Готово. Права уже настроены раньше (политика «profiles update own» разрешает
   каждому пользователю менять свой профиль, включая новые колонки; «profiles
   select all» — читать чужие, чтобы фото и контакты были видны участникам).

После этого «Телефон» и «Фото» будут сохраняться как обычно.

## Про фото

- Перед сохранением фото **сжимается** в браузере (canvas → JPEG, макс. 320×400 px),
  поэтому в базе лежит лёгкая картинка (обычно 20–50 КБ), а не тяжёлый оригинал.
- Хранится прямо в колонке `avatar` таблицы `profiles` как data-URL — отдельное
  файловое хранилище (Storage-бакет) не нужно.
