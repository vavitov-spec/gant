# Настройка облака: история версий и мгновенная синхронизация

Эту настройку нужно выполнить **один раз**. Она включает:
1. **Историю версий** — автоматические резервные копии всех графиков в облаке
   (последние 30 снимков, вкладка «Настройки → Резервные копии»).
2. **Мгновенную синхронизацию** — правки с одного устройства появляются на других
   за пару секунд (без неё обновление происходит раз в 20 секунд — тоже работает).

## Как выполнить

1. Откройте https://supabase.com/dashboard и войдите в свой аккаунт.
2. Выберите проект **vrgjczbtnomqwlppszpe**.
3. Слева в меню нажмите **SQL Editor** (иконка с надписью SQL).
4. Нажмите **New query**, вставьте целиком текст ниже и нажмите **Run** (или Ctrl+Enter).

```sql
-- 1. Таблица истории версий (резервные копии)
create table if not exists public.app_state_history (
  id bigint generated always as identity primary key,
  created_at timestamptz not null default now(),
  data jsonb not null
);

alter table public.app_state_history enable row level security;

drop policy if exists "history select" on public.app_state_history;
drop policy if exists "history insert" on public.app_state_history;
drop policy if exists "history delete" on public.app_state_history;

create policy "history select" on public.app_state_history for select using (true);
create policy "history insert" on public.app_state_history for insert with check (true);
create policy "history delete" on public.app_state_history for delete using (true);

-- 2. Мгновенная синхронизация (realtime-уведомления об изменениях)
alter publication supabase_realtime add table public.app_state;
```

5. Внизу должно появиться «Success. No rows returned». Готово.
6. Обновите страницу графиков — в «Настройки → Резервные копии» появится список копий
   (первая копия создастся автоматически после первого сохранения).

## Если что-то пошло не так

- Ошибка `relation "app_state" is already member of publication` на шаге 2 — это нормально,
  значит синхронизация уже была включена. Всё в порядке.
- Любая другая ошибка — скопируйте её текст и попросите Claude разобраться.

## Замечание о безопасности

Сейчас читать и изменять данные может любой, у кого есть адрес страницы (защита паролем «555»
работает только на самой странице). История версий страхует от потери данных, но настоящую
защиту даст вход по email-паролю (Supabase Auth) — код входа в приложении уже есть,
включение отложено. Когда решите включить — попросите Claude.
